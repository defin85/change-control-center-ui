## ADDED Requirements
### Requirement: Product-Test Runbook And Proof Path
The system SHALL document and verify the repo-owned product-test preparation and recovery workflow.

#### Scenario: Contributor prepares a product test from a clean checkout
- **WHEN** a contributor follows the documented product-test runbook
- **THEN** the repo provides one approved path to seed, reset, launch, and inspect the product-test shell
- **AND** the verification and docs surface stays aligned with the supporting automation
